Thermo Mining Control Plane MVP

Recommended single-page console for chat planning, plan review, active run monitoring, and artifact lookup

Desktop Shell
Thermo Mining Platform | Active run: one at a time | Settings | Health
Chat
Natural-language prompt history and assistant explanations
Structured intent summary generated from the latest chat turn
"Scan /mnt/disk3/raw and prepare paired reads for the default pipeline"
Plan Review
Bundle type, selected inputs, output root
Stage order and key parameter overrides
Editable fields: output root, resume, small threshold overrides
Run Monitor
Status: running | active stage | tmux session
Recent log lines and last error summary
Actions: stop, terminate, resume
Artifacts and History
summary.md | shortlist tables | execution_plan.json | runtime_state.json | stage logs
Recent completed runs and artifact paths
File Browser Modal
Browse any absolute path | search paths | scan bundles | pair FASTQ
Hard confirmation before plan submission for selected server paths
Primary Flow
1. Chat collects intent
2. File browser returns structured path selections
3. Planner emits ExecutionPlan
4. User edits a few fields in Plan Review
5. Confirm starts one tmux-backed run
6. Monitor and Artifacts read runtime_state.json + run directory
Boundary Decisions Locked In
Single user, no login system
One active run at a time
No local file upload; server-path selection only
Filesystem may browse any absolute path, but selection is type-checked and confirmed
Chat does planning, status queries, and basic failure explanation only
